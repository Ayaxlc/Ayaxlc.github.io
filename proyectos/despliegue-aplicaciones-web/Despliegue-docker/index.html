Despliegue de aplicación web con Docker

Este proyecto muestra cómo desplegar una aplicación web sencilla utilizando Docker y Docker Compose, separando el servidor web (Nginx) y el lenguaje de servidor (PHP) en contenedores independientes.

Objetivo del proyecto

El objetivo es aprender cómo se puede ejecutar una aplicación web sin instalar manualmente Nginx o PHP en el sistema operativo, utilizando contenedores Docker.

¿Cómo funciona?

El proyecto utiliza dos contenedores:

  • Nginx: atiende las peticiones del navegador.
  • PHP: ejecuta el código del servidor.

Ambos contenedores se gestionan mediante Docker Compose, lo que permite levantarlos de forma conjunta con un solo comando.

Estructura del proyecto

Despliegue-docker/
├── nginx/
│   └── default.conf
├── php/
│   └── Dockerfile
├── src/
│   └── index.php
├── docker-compose.yml
└── README.md

Ejecución en local

Para ejecutar el proyecto en local es necesario tener Docker instalado y ejecutar:

docker compose up -d --build

La aplicación estará disponible en:

http://localhost:8080

(Este puerto funciona únicamente en local).

Nota sobre GitHub Pages

Este proyecto se muestra en GitHub Pages únicamente a nivel descriptivo, ya que GitHub Pages no permite ejecutar PHP ni contenedores Docker. El despliegue real debe realizarse en local.
